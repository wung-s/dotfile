# TDD Skill

When implementing a feature with TDD:

## Phase 1 - RED

1. Create compilable stubs/skeletons for any new methods (empty implementations that return zero values)
2. Write test cases that call these stubs
3. Run tests and CONFIRM they compile and FAIL with expected assertion errors
4. Show the user the failing test output before proceeding

## Phase 2 - GREEN

5. Implement the minimum code to make tests pass
6. Run tests and confirm they PASS
7. Show the user the passing test output

## Phase 3 - REFACTOR

8. Clean up code if needed, re-run tests
   NEVER skip to Phase 2 without completing Phase 1.
